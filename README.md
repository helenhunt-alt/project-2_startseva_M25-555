Primitive DB
Учебный проект: консольное приложение — примитивная база данных.

Реализует базовые операции управления таблицами и хранит метаданные в JSON-файле.

Проект разработан в рамках выполнения учебного задания.

Описание проекта
Программа представляет собой простую файловую базу данных с возможностью:

создания таблиц;

просмотра списка таблиц;

удаления таблиц.

Информация о таблицах (их имена и структура столбцов) хранится в одном файле db_meta.json.

Работа с базой данных осуществляется через интерактивный консольный интерфейс.

Требования
Python 3.10+

Poetry

Установка и запуск
Установка зависимостей
bash
poetry install
Сборка и установка пакета
bash
poetry build
pip install dist/*.whl
Запуск программы
После установки пакета в терминале доступна команда:

bash
database
Управление таблицами
После запуска программы пользователь попадает в интерактивный режим.

Процесс работы с таблицей
Функции:

<command> create_table <имя_таблицы> <столбец1:тип> <столбец2:тип> .. — создать таблицу

<command> list_tables — показать список всех таблиц

<command> drop_table <имя_таблицы> — удалить таблицу

<command> exit — выход из программы

<command> help — справочная информация

Пример приглашения:

text
>>>Введите команду:
Поддерживаемые типы данных
Поддерживаются следующие типы столбцов:

int

str

bool

Создание таблицы
Команда:

text
create_table <имя_таблицы> <столбец1:тип> <столбец2:тип> ...
Особенности:

Если столбец ID:int не указан пользователем, он добавляется автоматически.

ID является уникальным ключом и всегда первым столбцом таблицы.

Если таблица с таким именем уже существует, создание невозможно.

Пример:

text
>>>Введите команду: create_table users name:str age:int is_active:bool
Просмотр списка таблиц
Команда:

text
list_tables
Пример:

text
>>>Введите команду: list_tables
users
products
Удаление таблицы
Команда:

text
drop_table <имя_таблицы>
Пример:

text
>>>Введите команду: drop_table users
Если таблица не существует, выводится сообщение об ошибке.

Обработка ошибок
При вводе некорректной команды выводится сообщение об ошибке.

При указании неподдерживаемого типа данных таблица не создаётся.

Все ошибки обрабатываются без аварийного завершения программы.
